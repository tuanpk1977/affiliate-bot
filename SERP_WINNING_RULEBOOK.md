# SERP Winning Rulebook

## Purpose

This document is the permanent knowledge base for future Codex and Copilot sessions working on Smile AI Review Hub content.

Before generating or revising any SEO article, the session must read:

1. PROJECT_GUIDE.md
2. This rulebook

The goal is simple: generate articles that are not merely readable, but are more likely to win search visibility, satisfy user intent, build trust, and convert readers.

This is a documentation-only rulebook. No code, no implementation, and no module changes are part of this document.

---

## Mandatory operating principle

Every future content session must follow this order:

1. Read PROJECT_GUIDE.md
2. Read this rulebook
3. Identify the target keyword and intent
4. Analyze the SERP and competitors
5. Build a stronger outline than the current competition
6. Write the article
7. Evaluate it with the scoring matrix
8. Improve weak sections
9. Add trust, schema, links, and CTA support
10. Only then prepare the article for publishing

If a session skips these steps, the output is considered incomplete.

## 1. Search Intent Rules

Search intent is the first decision point. The writer must identify the dominant intent before writing.

### 1.1 Informational intent

Use this when the user wants to learn, understand, or research.

Characteristics:
- the query is question-led or education-led
- the user wants explanation, examples, or guidance
- the article should prioritize clarity, depth, and usefulness

Required structure:
- clear answer near the top
- explain the concept simply
- include examples, steps, or definitions where useful
- avoid heavy sales pressure

Good examples:
- what is X
- how does X work
- how to use X
- beginner guide

### 1.2 Commercial intent

Use this when the user is evaluating options before buying.

Characteristics:
- the user is researching products or services
- the user wants comparisons, strengths, weaknesses, and fit guidance
- the user may still be undecided

Required structure:
- comparison framework
- decision criteria
- practical pros and cons
- trust and evidence signals
- clear next-step guidance

### 1.3 Transactional intent

Use this when the user is close to purchase or wants a direct action.

Characteristics:
- terms like buy, pricing, plans, free trial, best deal, sign up, review
- the user wants an answer that supports a decision now

Required structure:
- clear pricing or value section
- direct explanation of what the product does best
- recommendation summary
- CTA that feels helpful rather than pushy

### 1.4 Navigational intent

Use this when the user is looking for a specific destination.

Characteristics:
- the query points to a known brand, product, or page
- the user may already know the name

Required structure:
- direct, factual information
- brand-specific details
- comparison or feature context if relevant
- clear page purpose and next steps

### 1.5 Intent rule of thumb

The article should always answer the user’s likely reason for searching, not just repeat the keyword.

## 2. Article Type Decision Matrix

Choose the article type based on search intent and user journey.

### 2.1 Review

Use for:
- product evaluation queries
- “is X worth it”
- “X review”
- “best X for Y” where the article is evaluative

Required elements:
- a clear verdict near the top
- strengths and weaknesses
- use cases and audience fit
- transparency and evidence
- recommendation section

### 2.2 Comparison

Use for:
- “X vs Y”
- “best X alternatives”
- “which tool is better”

Required elements:
- direct side-by-side comparison
- criteria-based analysis
- decision guidance for different audiences
- comparison table where appropriate
- practical recommendation

### 2.3 Alternatives

Use for:
- “best alternatives to X”
- “other options instead of X”

Required elements:
- explain alternatives clearly
- show when each alternative is better
- avoid vague listicles
- include fit-based decision logic

### 2.4 Best list

Use for:
- “best X”
- “top 10 X”
- “best tools for Y”

Required elements:
- explain ranking logic
- define evaluation criteria
- give reasons for each inclusion
- avoid arbitrary or promotional ordering

### 2.5 Tutorial

Use for:
- “how to use X”
- “step-by-step X”
- “guide to X”

Required elements:
- practical, sequential structure
- numbered steps or clear phases
- actionable language
- examples or screenshots if helpful

### 2.6 Pricing

Use for:
- “X pricing”
- “how much does X cost”
- “X plans”

Required elements:
- clarity around pricing tiers
- value explanation
- affordability or ROI framing
- note limitations or feature differences

### 2.7 Vs

Use for:
- direct competitor contrasts
- “X vs Y” with a head-to-head emphasis

Required elements:
- direct comparison framing
- concise verdict section
- highlight best use cases
- show tradeoffs clearly

### 2.8 Use cases

Use for:
- “X use cases”
- “when to use X”
- “who should use X”

Required elements:
- scenario-based structure
- practical examples
- audience fit guidance
- benefits and limitations by use case

### 2.9 Article type decision rule

The writer should choose the article type that best matches the likely user need, not simply the most convenient format.

## 3. CTR Optimization Rules

CTR should be improved before writing, not treated as a late-stage decoration.

### 3.1 Title scoring

Every title candidate should be scored using these dimensions:

- relevance to the keyword
- intent match
- clarity
- curiosity
- buyer intent alignment
- commercial usefulness
- uniqueness versus competitors
- emotional strength
- trustworthiness
- length appropriateness

Suggested weighting:
- relevance: 25%
- intent alignment: 20%
- clarity: 15%
- curiosity: 15%
- buyer intent: 10%
- commercial value: 10%
- uniqueness: 5%

### 3.2 Curiosity gap

A strong title should create a reason to click without being misleading.

Good curiosity patterns:
- a surprising outcome
- a hidden tradeoff
- a practical result
- a tension between options
- a specific promise

Avoid:
- empty hype
- fake secrets
- sensationalism without substance

### 3.3 Emotional triggers

Titles should trigger a useful emotion.

Examples:
- relief: save time, avoid mistakes
- confidence: choose wisely, avoid regret
- assurance: trusted, tested, proven
- curiosity: what most people miss
- convenience: simple, beginner-friendly

### 3.4 Numbers

Numbers help when they genuinely organize the article.

Use numbers for:
- list-style content
- comparisons
- step-by-step guides
- top-X roundups
- mistakes or lessons

Do not force numbers where they do not help the structure.

### 3.5 Year

Use the current year when it adds true freshness.

Good use cases:
- current year roundups
- current pricing or feature references
- trend-aware comparisons

Do not use the year as a filler keyword.

### 3.6 Power words

Use power words sparingly and honestly.

Useful categories:
- clarity: practical, reliable, proven
- urgency: now, fast, simple
- confidence: best, top, trusted
- value: affordable, efficient, smart
- curiosity: overlooked, surprising, essential

### 3.7 A/B headline principles

Before choosing a final title, generate multiple versions.

At minimum, include:
- a baseline title
- a benefit-first title
- a curiosity title
- a buyer-intent title
- a comparison title

The winning title should balance clarity, relevance, and click appeal.

## 4. SERP Coverage Rules

The article must be designed to outperform the current ranking pages, not simply mimic them.

### 4.1 Analyze top competitors

Before writing, the session should inspect the top results for the target query.

Look for:
- article structure
- heading patterns
- FAQ presence
- comparison table presence
- pricing block presence
- schema usage
- trust signals
- CTA style
- content depth

### 4.2 Identify missing sections

The article should cover what competitors miss or cover weakly.

Common gap opportunities:
- missing FAQ
- weak comparison structure
- missing pricing clarity
- weak buying guide
- missing entities or supporting context
- weak depth on use cases
- missing transparency or limitations
- weak trust signals

### 4.3 Build a better outline

The outline should be stronger than the top results.

The outline should include:
- the direct answer near the top
- the most useful subtopics for the intent
- comparison or decision-support sections when relevant
- FAQ section
- limitations and transparency section
- CTA placement
- internal link placement

### 4.4 Content gap scoring

A gap score should be assigned to each opportunity area.

High-priority gaps:
- missing or weak FAQ
- missing comparison logic
- missing buying guidance
- weak pricing information
- weak CTA usefulness
- weak trust elements
- weak EEAT support

The writer should prioritize solving the biggest content gaps first.

## 5. EEAT Rules

EEAT is a trust and credibility framework. It must be visible in the article structure.

### 5.1 Author credibility

The article should communicate who is evaluating the topic and why the evaluation is credible.

Include:
- author or editorial identity
- relevant subject experience
- transparency about review basis
- clear editorial standards

### 5.2 Testing methodology

If the article evaluates a product, service, or tool, the testing methodology should be clear.

It should explain:
- what was tested
- how it was tested
- how long it was tested
- what criteria mattered
- what conditions were used

### 5.3 Screenshots

Screenshots should be used where they support clarity or evidence.

Use them for:
- feature demonstrations
- interface examples
- comparison evidence
- results or workflow steps

### 5.4 Pros and cons

The article should show a balanced view.

Include:
- concrete strengths
- concrete weaknesses
- realistic tradeoffs
- audience fit explanation

### 5.5 Verdict

Every review or comparison should end with a clear verdict.

The verdict should answer:
- who should choose this option
- who should avoid it
- whether it is worth the cost
- what the strongest use case is

### 5.6 Transparency

The article must be transparent about limitations, uncertainty, and context.

Do not:
- overstate certainty
- present opinions as fact
- hide weaknesses
- make claims without support

### 5.7 Affiliate disclosure

If the article includes affiliate or commercial intent, disclosure must be clear and straightforward.

The disclosure should explain:
- that the site may earn commissions
- that recommendations are editorially guided
- that the content remains focused on usefulness and honesty

## 6. Internal Linking Rules

Internal links should be planned, not random.

### 6.1 Hub pages

Hub pages should group related content into a central topic page.

They should:
- summarize the topic clearly
- connect to supporting articles
- provide a route into the cluster
- help the reader continue exploring

### 6.2 Cluster pages

Cluster pages should reinforce topical depth.

They should connect to:
- the hub page
- adjacent support articles
- related reviews or comparisons
- relevant buying-guide or FAQ content

### 6.3 Money pages

Money pages should receive contextual support from related content.

Support them with:
- review pages
- comparison pages
- tutorials
- FAQ pages
- buying guides

### 6.4 Comparison pages

Comparison pages should link to the most relevant alternatives, reviews, and buying-guide pages.

### 6.5 Anchor diversity

Anchor text should be varied and natural.

Use a mix of:
- descriptive anchors
- partial-match anchors
- branded anchors
- generic anchors where appropriate

Avoid repetitive keyword stuffing.

### 6.6 Contextual links

Links should appear where they are useful to the reader.

Good placement:
- within relevant paragraphs
- near supporting examples
- in “related reads” or “next steps” sections
- in comparison and verdict sections

---

## 7. Affiliate Conversion Rules

The article should help the reader decide while remaining useful and trustworthy.

### 7.1 CTA placement

CTA placement should be deliberate.

Good placement:
- after the verdict or recommendation
- after a comparison or pricing block
- after a strong decision-support section
- near a clear next step

Avoid placing CTA too early or too aggressively in a way that breaks the reading flow.

### 7.2 CTA wording

CTA wording should be helpful and non-spammy.

Good approaches:
- recommend the best fit
- offer a clear next step
- explain why the option is useful
- support decision-making

Avoid:
- aggressive hype
- forced urgency without value
- generic sales language

### 7.3 Comparison tables

Comparison tables improve decision support.

They should clarify:
- features
- pricing differences
- target audience fit
- strengths and weaknesses
- overall recommendation

### 7.4 Trust elements

Trust elements make conversion feel safer.

Include:
- transparent methodology
- honest limitations
- balanced pros and cons
- clear verdict
- evidence-based language

### 7.5 Pricing blocks

Pricing blocks should be clear and useful.

They should explain:
- what is included
- what the differences are
- whether value is strong
- if there are tradeoffs or limitations

### 7.6 FAQs

FAQs should answer the most likely objections and questions.

Good FAQ topics:
- who is it best for
- what are the downsides
- how does it compare
- is it worth the cost
- what should the reader know before buying

## 8. Schema Rules

Schema should support the content type and make the page easier for search engines to interpret.

### 8.1 Review schema

Use when the article is a review or evaluation.

Include support for:
- review details
- author or publisher context
- item being reviewed
- summary or verdict where appropriate

### 8.2 FAQ schema

Use when the article includes a structured FAQ section.

Each FAQ item should be concise, factual, and useful.

### 8.3 Product schema

Use when the article is centered on a product or tool.

Include product-level context where appropriate.

### 8.4 Breadcrumb schema

Use when the page sits within a content hierarchy.

It should reflect the relationship of the page to its cluster or category.

### 8.5 Organization schema

Use when the article or site has a clear organizational identity.

It should reflect the publisher or editorial entity clearly and consistently.

### 8.6 Schema rule

Schema must be accurate, relevant, and aligned with the content. It should support the page, not feel decorative.

## 9. Publishing Checklist

Before publishing, confirm the following:

- the target intent is clear
- the article type matches the intent
- the title was scored and chosen carefully
- the outline is stronger than competing pages
- the article covers missing information and gaps
- the article is clear, helpful, and structured
- trust and EEAT elements are present
- pricing or comparison sections are included when relevant
- FAQs address likely objections
- internal links are relevant and contextual
- disclosure is present where needed
- schema is aligned with the content type
- the CTA feels natural and useful
- the article passes the quality thresholds

If any major item is missing, the article should not be published yet.

## 10. Content Scoring Matrix

Use this matrix to score the article before publication.

### 10.1 Coverage

How complete is the topic?

Score high when:
- the main subtopics are covered
- the article answers likely follow-up questions
- the page feels comprehensive

### 10.2 Intent

How well does the article match the search intent?

Score high when:
- the structure fits the user need
- the article answers the likely reason for the query
- the content is not misaligned

### 10.3 Authority

How strong is the topical depth and expertise?

Score high when:
- the article is specific, informed, and detailed
- the reasoning is strong
- the evidence and comparisons are useful

### 10.4 Trust

How trustworthy does the article feel?

Score high when:
- credibility is visible
- limitations are acknowledged
- disclosures are clear
- the article is balanced

### 10.5 Freshness

How current and timely is the content?

Score high when:
- updates are relevant
- pricing or features are current
- the article reflects recent context when needed

### 10.6 Readability

How easy is the content to scan and understand?

Score high when:
- paragraphs are concise
- headings guide the reader
- the structure is clear
- the article flows logically

### 10.7 Commercial value

How useful is the article for decision-making and conversion?

Score high when:
- the content supports buying decisions
- comparison and value sections are strong
- the CTA is helpful and relevant

### 10.8 CTR

How likely is the article to earn clicks from search results?

Score high when:
- the title is relevant and compelling
- the meta description reinforces the value
- the page promises a useful answer

### 10.9 Overall score

Overall score should reflect the combined strength of all the above categories.

## 11. Quality Thresholds

Use these thresholds as a decision guide.

### 11.1 Strong article

An article is strong when:
- it clearly matches the target intent
- it covers the topic more fully than the competition
- it includes trust and transparency elements
- it is structured for both readers and search engines
- it has a helpful CTA and useful internal links

### 11.2 Minimum acceptable bar

The article should not be published if:
- the intent is unclear
- the structure is shallow
- the content is generic
- the article lacks transparency or trust signals
- the CTA is weak or inappropriate
- the article feels obviously thin compared with top competitors

### 11.3 Preferred target

The preferred target is a high-quality article that is clear, useful, comprehensive, trustworthy, and conversion-ready.

## 12. Common Failure Cases

These are the most common ways content underperforms.

### 12.1 Weak intent match

The article is written for the wrong user need.

### 12.2 Keyword stuffing

The article repeats the target term awkwardly instead of serving the user.

### 12.3 Shallow coverage

The article answers the headline but leaves out important subtopics, questions, or decision factors.

### 12.4 Generic title

The title is too broad, too bland, or too similar to competitors.

### 12.5 Missing trust signals

The article reads promotional, vague, or unconvincing.

### 12.6 Weak comparison structure

The article does not explain how the options differ or where each one fits.

### 12.7 Poor CTA design

The CTA is too pushy, too early, or not useful.

### 12.8 Weak internal linking

The page stands alone and lacks connection to the broader site architecture.

### 12.9 Missing schema or structured support

The article is good but not fully supported for search presentation.

---

## 13. Recovery Strategy

If an article is weak, recover it before publishing.

### 13.1 Identify the weak point

Ask:
- is the intent wrong
- is the outline too thin
- is the article missing trust elements
- is the comparison too weak
- is the CTA misplaced
- is the structure too generic

### 13.2 Improve the highest-impact section first

Do not rewrite everything blindly. Improve the section that most weakens the article.

### 13.3 Add missing coverage

Add the content the SERP and user likely need.

### 13.4 Strengthen trust and conversion

Add the trust, disclosure, and decision-support elements the article lacks.

### 13.5 Re-score before publishing

The article should only move forward once the weak points have been addressed.

---

## 14. Continuous Improvement Rules

The content system should improve over time.

### 14.1 Learn from results

Review which article types perform best and which patterns win more often.

### 14.2 Compare against actual SERP winners

Use the current top results as a benchmark, not as a template to imitate blindly.

### 14.3 Improve article patterns

Refine the structure, title principles, trust elements, CTA approach, and internal linking logic over time.

### 14.4 Keep quality high

Do not trade quality for speed. A stronger article beats a faster but weaker one.

### 14.5 Keep the process repeatable

Future sessions should be able to follow the same process consistently and produce better content with each iteration.

## 15. Daily Workflow for Future AI Sessions

Every future content session should follow this workflow daily.

### Step 1: Read the guide
Read PROJECT_GUIDE.md first to understand the project boundaries, site goals, and publishing context.

### Step 2: Read the rulebook
Read this document before drafting any content so the session follows the project’s quality and SEO standards.

### Step 3: Define the target query
Identify the topic, keyword, and likely search intent.

### Step 4: Determine the article type
Choose review, comparison, alternatives, best list, tutorial, pricing, vs, or use cases based on intent.

### Step 5: Analyze the SERP
Review the current top results and identify what they cover well and where they are weak.

### Step 6: Build the keyword cluster
Map the parent topic, supporting topics, semantic terms, buyer terms, informational terms, and comparison terms.

### Step 7: Create the outline
Write a stronger outline than the competing pages.

### Step 8: Draft the article
Write the article with the correct intent, structure, and depth.

### Step 9: Score the draft
Evaluate the content against the coverage, intent, authority, trust, readability, CTR, and commercial value criteria.

### Step 10: Improve weak sections
Revise the article where it is thin, unclear, or weak.

### Step 11: Add trust and conversion support
Add EEAT elements, disclosure, comparison support, pricing clarity, FAQ, and CTA where appropriate.

### Step 12: Add internal links and schema support
Connect the article to the broader site structure and prepare the appropriate schema support.

### Step 13: Run final audit
Confirm the article is ready for publication.

### Step 14: Publish only if the quality bar is met
The article should not be published if it fails the major quality checks.
