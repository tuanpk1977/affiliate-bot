# SEO Autonomous Writer Design

## Objective

The autonomous SEO writer should act as a guided content system that prepares, writes, evaluates, and improves articles before anything is published. It should not write blindly. Instead, it should first understand the project rules, the target intent, the competitive environment, and the content gaps, then generate an article that is more complete, more useful, and more likely to perform well in search.

This document is planning-only. No implementation work, no code, and no publishing actions are included.

---

## Core principle

Before writing any article, the system must complete a structured preparation workflow:

1. Read project guide
2. Read quality rules
3. Identify search intent
4. Cluster keywords
5. Analyze top competitors
6. Detect missing information
7. Generate better outline
8. Generate article
9. Run quality gate
10. Improve weak sections
11. Generate schema
12. Generate internal links
13. Generate affiliate CTA
14. Run final audit
15. Only then publish

Each stage should be treated as a decision point rather than a simple generation step.

---

## 1. Read project guide

### Purpose

The writer must first understand the project’s rules, constraints, and operating boundaries before creating any content.

### What it should review

- content goals
- publishing boundaries
- tone and voice requirements
- site structure expectations
- compliance and disclosure expectations
- topic and affiliate strategy
- documentation constraints

### Output

A short operating context summary that informs the rest of the workflow.

### Why this matters

If the writer does not understand the project context, it may produce content that is misaligned with the site’s goals, style, or publishing rules.

---

## 2. Read quality rules

### Purpose

The system must understand what “good” means for this project before it writes.

### What it should review

- quality expectations
- trust and EEAT expectations
- formatting rules
- article structure preferences
- CTA rules
- keyword usage rules
- readability expectations
- factuality and transparency rules

### Output

A quality checklist that the article will be judged against.

### Why this matters

Quality rules prevent the writer from generating shallow, promotional, or low-value content.

---

## 3. Identify search intent

### Purpose

The system must determine what the user likely wants when searching the target query.

### Supported intents

- Review
- Comparison
- Pricing
- Alternatives
- Tutorial
- Best
- Vs
- Use cases
- FAQ

### What the system should determine

- the main intent behind the keyword
- the likely user journey
- whether the person is researching, comparing, deciding, or buying
- the expected article format

### Output

An intent profile with:

- intent label
- confidence score
- article format recommendation
- primary content goals

### Why this matters

A page that matches search intent is more likely to satisfy the query and earn better engagement.

---

## 4. Cluster keywords

### Purpose

The system should build a topical map before drafting so the article can cover related concepts rather than only the main keyword.

### Keyword clusters should include

- Parent topic
- Supporting topics
- Semantic keywords
- Buyer keywords
- Informational keywords
- Comparison keywords

### Output

A keyword cluster brief that identifies:

- the main theme
- related subtopics
- probable questions readers have
- decision-support terms
- comparison-related language

### Why this matters

Keyword clustering helps the writer create a page that is broader, deeper, and more useful than a narrow keyword-only article.

---

## 5. Analyze top competitors

### Purpose

The writer should understand what currently ranks before building a stronger page.

### What it should inspect

- top search results for the target query
- title patterns
- structure patterns
- section coverage
- FAQ usage
- schema usage
- internal linking patterns
- trust and authority signals
- commercial angle and CTA patterns

### Output

A competitor intelligence report that includes:

- common article structure
- what competitors cover well
- what they miss
- which gaps are likely to produce a better page

### Why this matters

A strong article should not be written in isolation. It should be designed to outperform the current leading results.

---

## 6. Detect missing information

### Purpose

The system should identify what the current competitive landscape is not covering well.

### Missing information categories

- missing topics
- weak FAQ
- weak buying guide
- weak comparison
- weak pricing
- missing entities
- weak trust signals
- weak CTA quality
- missing schema opportunities
- weak internal links
- weak EEAT signals

### Output

A gap analysis report that highlights:

- what should be added
- what should be expanded
- what should be clarified
- where the article can become more useful than competitors

### Why this matters

The article should win by being more complete, more helpful, and more decision-ready than the results it competes against.

---

## 7. Generate better outline

### Purpose

The writer should build a stronger article structure before drafting.

### Outline should reflect

- the identified intent
- the keyword cluster
- the competitor analysis
- the missing information gaps
- the desired trust and conversion signals

### Outline should include

- hero introduction
- key answer or verdict
- comparison or evaluation sections
- FAQs
- pricing or value section if relevant
- limitations and transparency
- CTA section
- internal link placement plan

### Output

A detailed article outline with section priorities and content goals.

### Why this matters

A strong outline makes the final article more coherent, deeper, and more likely to satisfy both users and search intent.

---

## 8. Generate article

### Purpose

The system should draft the article using the outline, intent, quality rules, and gap analysis.

### Article generation should prioritize

- helpfulness
- clarity
- directness
- trustworthiness
- relevance
- topical depth
- decision support
- natural commercial usefulness

### Output

A first draft article with clear sections, strong structure, and the intended content depth.

### Why this matters

The first draft should already be shaped by research and structure rather than generated as a generic block of text.

---

## 9. Run quality gate

### Purpose

The draft should be scored before it is considered final.

### Quality gate should evaluate

- search intent match
- topical coverage
- readability
- trust signals
- CTA strength
- EEAT quality
- structure quality
- comparison or buying-guide usefulness
- freshness and relevance
- schema readiness

### Output

A quality scorecard with strengths, weaknesses, and improvement recommendations.

### Why this matters

The system should not assume the first draft is sufficient just because it is written.

---

## 10. Improve weak sections

### Purpose

The writer should revise the draft where the quality gate identifies weak areas.

### Improvement focus

- add missing sections
- strengthen FAQ coverage
- improve clarity or structure
- add evidence or trust signals
- improve comparison depth
- improve buying guidance
- strengthen CTA quality
- improve readability and scannability

### Output

An improved draft with targeted refinements rather than a full rewrite.

### Why this matters

A strong writer should be able to revise weak areas rather than stop at a mediocre first pass.

---

## 11. Generate schema

### Purpose

The article should be prepared with structured data that supports search visibility and richer presentation.

### Schema types may include

- Article
- Review
- FAQ
- Breadcrumb
- SoftwareApplication
- Organization
- Person
- Product

### Output

A structured data recommendation bundle aligned with the content type.

### Why this matters

Schema helps search engines better understand the content and can enhance how the page is represented in search results.

---

## 12. Generate internal links

### Purpose

The article should be connected into the broader site architecture.

### Internal linking should support

- related articles
- topic clusters
- hub pages
- money pages
- review pages
- comparison pages

### Output

A recommended internal link plan with source pages, target pages, and anchor suggestions.

### Why this matters

Good internal linking improves topical relevance, content discoverability, and site authority distribution.

---

## 13. Generate affiliate CTA

### Purpose

The article should include a commercial call to action that is useful, natural, and aligned with the article’s intent.

### CTA design should consider

- reader stage
- article intent
- trust requirements
- disclosure expectations
- conversion goals
- non-spammy tone

### Output

A recommended CTA block that is useful and appropriate for the content and audience.

### Why this matters

The article should be commercially useful without feeling forced or deceptive.

---

## 14. Run final audit

### Purpose

Before publishing, the article should be checked as a complete package.

### Final audit should verify

- quality gate score meets expectations
- search intent is matched
- structure is coherent
- disclosures are present
- internal links are appropriate
- schema is reasonable
- CTA is appropriate
- content is balanced and useful
- the page is ready to publish

### Output

A final readiness report with pass/fail guidance.

### Why this matters

The final audit ensures the article is not only written, but also ready for publication in a responsible and effective way.

---

## 15. Only then publish

### Purpose

Publishing should be treated as the last stage in a controlled workflow, not the first.

### Publishing rule

The article should only be published if the preparation and audit stages are complete and the article satisfies the project’s quality bar.

### Why this matters

A publishing gate prevents low-value or poorly prepared content from being released into the site.

## Expected outcome

An autonomous SEO writer designed in this way should be capable of generating content that is:

- more relevant to the target query
- more structured and useful
- more competitive against existing search results
- stronger in trust and EEAT signals
- better connected into the site architecture
- more ready for publication with less manual cleanup
