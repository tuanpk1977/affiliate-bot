AI Coding Workflow Checklist

Use this before publishing any AI-assisted page or project update.

  1. Use Windsurf to bootstrap the rough project structure.
  2. Use Codex to fix/refine logic, architecture, build, and deploy issues.
  3. Check language integrity for English and /vi/ pages.
  4. Check SEO title, meta description, H1, canonical, and hreflang.
  5. Build the sitemap and keep /go/ pages out of it.
  6. Sync the static output to docs for GitHub Pages.
  7. Submit the sitemap to Google Search Console.
  8. Create social drafts for manual review.
  9. Track results with UTM links and tracking maps.
  10. Improve content quality based on issues and data.

FAQ

Is the checklist free?

Yes. It is a free workflow checklist for reviewing AI-assisted coding work before publishing.

Does the form automatically store my email?

Not yet. The site is static on GitHub Pages, so the current form uses a mailto fallback while subscribers.csv is prepared for a future integration.

Does the checklist replace manual review?

No. It helps you remember what to check, but code, SEO, sitemap, and content still need human review.
